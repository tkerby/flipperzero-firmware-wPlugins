# FAM (манифесты приложений Flipper) {#app_manifests}

Все компоненты прошивки Flipper Zero — сервисы, пользовательские приложения и системные настройки — разрабатываются независимо. Каждый компонент имеет файл манифеста системы сборки с именем `application.fam`, который определяет основные свойства этого компонента и его связи с другими частями системы.

При сборке прошивки `fbt` собирает все манифесты приложений и обрабатывает их зависимости. Затем он строит только те компоненты, которые указаны в текущей конфигурации сборки. Подробнее о конфигурациях сборки см. в [документации FBT](fbt.md).

## Определение приложения

Свойства компонента прошивки объявляются во фрагменте кода Python, формирующем вызов функции `App()` с различными параметрами.

Обязательны только два параметра: **appid** и **apptype**. Остальные необязательны и могут иметь смысл только для определённых типов приложений.

### Параметры

- **appid**: строка, ID приложения в системе сборки. Используется для указания, какое приложение включить в конфигурацию сборки, а также для разрешения зависимостей и конфликтов.

- **apptype**: элемент перечисления FlipperAppType.\*. Допустимые значения:

| Элемент перечисления | Тип компонента прошивки                                                              |
| -------------------- | ------------------------------------------------------------------------------------ |
| SERVICE              | Системный сервис, создаётся на раннем этапе запуска                                  |
| SYSTEM               | Приложение не отображается в меню. Может быть запущено другими приложениями или из CLI |
| APP                  | Обычное приложение для главного меню                                                 |
| PLUGIN               | Приложение, которое должно быть включено в прошивку и размещено в меню Plugins       |
| DEBUG                | Приложение, видимое только в меню Debug при включённом режиме отладки                |
| ARCHIVE              | Единственное приложение Archive                                                      |
| SETTINGS             | Приложение для размещения в меню системных настроек                                  |
| STARTUP              | Функция обратного вызова для запуска при старте системы. Не определяет отдельное приложение |
| EXTERNAL             | Приложение, которое должно быть собрано как плагин `.fap`                            |
| METAPACKAGE          | Не определяет код для запуска, используется для объявления зависимостей и пакетов приложений |

- **name**: имя, отображаемое в меню.
- **entry_point**: C-функция, используемая как точка входа приложения. Обратите внимание, что имена функций C++ подвергаются манглингу, поэтому для использования их как точек входа нужно обернуть их в `extern "C"`.
- **flags**: внутренние флаги для системных приложений. Не используйте.
- **cdefines**: определения препроцессора C для глобального объявления в других приложениях, когда текущее приложение включено в активную конфигурацию сборки. **Для внешних приложений**: указанные определения используются при сборке самого приложения.
- **requires**: список ID приложений для включения в конфигурацию сборки, когда текущее приложение указано в списке приложений для сборки.
- **conflicts**: список ID приложений, с которыми текущее приложение конфликтует. Если какой-либо из них найден в сформированном списке приложений, `fbt` прервёт процесс сборки прошивки.
- **provides**: функционально идентично полю **_requires_**.
- **stack_size**: размер стека в байтах для выделения приложению при его запуске. Обратите внимание, что выделение слишком маленького стека для работы приложения вызовет системный сбой из-за переполнения стека, а выделение слишком большого объёма стека уменьшит размер доступной кучи для обработки данных приложениями. _Примечание: вы можете использовать команды CLI `top` и `free` для профилирования использования памяти вашим приложением._
- **icon**: анимированная иконка из встроенных ресурсов для использования при сборке приложения как части прошивки.
- **order**: порядок приложения в своей группе при сортировке элементов в ней. Чем меньше порядок, тем ближе к началу списка размещается элемент. _Используется для упорядочивания хуков запуска и элементов меню._
- **sdk_headers**: список C-заголовочных файлов из кода этого приложения для включения в определения API для внешних приложений.
- **targets**: список строк и имён целей, с которыми совместимо это приложение. Если не указано, приложение собирается для всех целей. Значение по умолчанию — `["all"]`.
- **resources**: имя папки в исходной папке приложения для упаковки ресурсов SD-карты для этого приложения. Они будут использованы только если приложение включено в конфигурацию сборки. Значение по умолчанию — `""`, что означает отсутствие упаковки ресурсов.

#### Параметры для внешних приложений

Следующие параметры используются только для [FAP](./AppsOnSDCard.md):

- **sources**: список строк, масок имён файлов для сбора исходников в папке приложения. Значение по умолчанию `["*.c*"]` включает исходные файлы C и C++. Приложения не могут использовать папку `"lib"` для своего исходного кода, так как она зарезервирована для **fap_private_libs**. Пути, начинающиеся с `"!"`, исключаются из списка источников. Они также могут включать подстановочные символы и имена каталогов. Например, значение `["*.c*", "!plugins"]` включит все исходные файлы C и C++ в папке приложения, кроме файлов в папках `plugins` (и `lib`). Пути без подстановочных символов (`*, ?`) трактуются как полные литеральные пути как для включения, так и для исключения.
- **fap_version**: строка, версия приложения. Значение по умолчанию — "0.1". Также можно использовать кортеж из 2 чисел в форме (x,y) для указания версии. Можно добавить больше разделённых точкой частей к версии, например номер патча, но в собранном .fap хранятся только мажорная и минорная версии.
- **fap_icon**: имя файла `.png`, 1-битная глубина цвета, 10x10px, для встраивания в файл `.fap`.
- **fap_libs**: список дополнительных библиотек для линковки с приложением. Предоставляет доступ к дополнительным функциям, которые не экспортируются как часть основной прошивки, ценой увеличения размера файла `.fap` и потребления ОЗУ.
- **fap_category**: строка, может быть пустой. Подкатегория приложения, также определяет путь FAP в папке apps файловой системы.
- **fap_description**: строка, может быть пустой. Краткое описание приложения.
- **fap_author**: строка, может быть пустой. Автор приложения.
- **fap_weburl**: строка, может быть пустой. Домашняя страница приложения.
- **fap_icon_assets**: строка. Если присутствует, определяет имя папки для сбора графических ресурсов для этого приложения. Эти изображения будут скомпилированы вместе с исходниками приложения. Подробнее см. [FAP assets](AppsOnSDCard.md).
- **fap_extbuild**: обеспечивает поддержку частей исходников приложения, собираемых внешними инструментами. Содержит список определений `ExtFile(path="имя файла", command="shell команда")`. `fbt` выполнит указанную команду для каждого файла в списке.
- **fal_embedded**: булево, по умолчанию `False`. Применяется только к типу PLUGIN. Если `True`, плагин будет встроен в .fap файл хост-приложения как ресурс и извлечён в папку `apps_assets/APPID` при его запуске. Это позволяет распространять плагины как часть хост-приложения.

Обратите внимание, что команды выполняются в корневой папке прошивки, а все промежуточные файлы должны быть размещены во временной папке сборки приложения. Для этого можно использовать подстановку шаблонов `fbt`: `${FAP_WORK_DIR}` будет заменён на путь к временной папке сборки приложения, а `${FAP_SRC_DIR}` — на путь к папке исходников приложения. Также можно использовать другие переменные, определённые внутри `fbt`.

Пример сборки приложения из Rust-исходников:

```python
    sources=["target/thumbv7em-none-eabihf/release/libhello_rust.a"],
    fap_extbuild=(
        ExtFile(
            path="${FAP_WORK_DIR}/target/thumbv7em-none-eabihf/release/libhello_rust.a",
            command="cargo build --release --verbose --target thumbv7em-none-eabihf --target-dir ${FAP_WORK_DIR}/target --manifest-path ${FAP_SRC_DIR}/Cargo.toml",
        ),
    ),
```

- **fap_private_libs**: список дополнительных библиотек, распространяемых как исходники вместе с приложением. Эти библиотеки будут собраны как часть процесса сборки приложения.
  Исходники библиотек должны быть размещены в подпапке папки `lib` в исходной папке приложения.
  Каждая библиотека определяется вызовом функции `Lib()`, принимающей следующие параметры:

  - **name**: имя папки библиотеки. Обязательно.
  - **fap_include_paths**: список относительных путей библиотеки для добавления в список путей включения родительского fap. Значение по умолчанию — `["."]`, что означает корень исходников библиотеки.
  - **sources**: список масок имён файлов для сбора заголовочных файлов для этой библиотеки. Пути относительны корня исходников библиотеки. Значение по умолчанию — `["*.c*"]`.
  - **cflags**: список дополнительных флагов компилятора для использования при сборке этой библиотеки. Значение по умолчанию — `[]`.
  - **cdefines**: список дополнительных определений препроцессора для использования при сборке этой библиотеки. Значение по умолчанию — `[]`.
  - **cincludes**: список дополнительных путей включения для использования при сборке этой библиотеки. Пути относительны корня приложения. Это можно использовать для предоставления внешних путей поиска для кода этой библиотеки — для заголовков конфигурации. Значение по умолчанию — `[]`.

Пример сборки приложения с приватной библиотекой:

```python
    fap_private_libs=[
            Lib(
                name="mbedtls",
                fap_include_paths=["include"],
                sources=[
                    "library/des.c",
                    "library/sha1.c",
                    "library/platform_util.c",
                ],
                cdefines=["MBEDTLS_ERROR_C"],
            ),
            Lib(
                name="loclass",
                cflags=["-Wno-error"],
            ),
        ],
```

Для этого фрагмента `fbt` соберёт 2 библиотеки: одну из исходников в папке `lib/mbedtls` и другую из исходников в папке `lib/loclass`. Для библиотеки `mbedtls` `fbt` добавит `lib/mbedtls/include` в список путей включения для приложения и скомпилирует только файлы, указанные в списке `sources`. Дополнительно `fbt` включит определение препроцессора `MBEDTLS_ERROR_C` для исходников `mbedtls`.
Для библиотеки `loclass` `fbt` добавит `lib/loclass` в список путей включения для приложения и соберёт все исходники в этой папке. Также `fbt` отключит трактовку предупреждений компилятора как ошибок для библиотеки `loclass`, что может быть полезно при компиляции больших сторонних кодовых баз.

Обе библиотеки будут слинкованы с приложением.

## Содержимое файла .fam

Файл `.fam` содержит одно или несколько определений приложений. Например, вот часть `applications/service/bt/application.fam`:

```python
App(
    appid="bt_start",
    apptype=FlipperAppType.STARTUP,
    entry_point="bt_on_system_start",
    order=70,
)

App(
    appid="bt_settings",
    name="Bluetooth",
    apptype=FlipperAppType.SETTINGS,
    entry_point="bt_settings_app",
    stack_size=1 * 1024,
    requires=[
        "bt",
        "gui",
    ],
    order=10,
)
```

Больше примеров см. в файлах `.fam` из различных частей прошивки.
